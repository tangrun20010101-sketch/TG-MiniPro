# 通兑经济系统设计

## 设计理念

- **货币体系**：三层结构，偏挖矿玩乐花时间体系
- **卡牌体系**：三层品质，偏抽奖幸运体系
- **通兑关系**：中间层（金币）可上下（与货币）、左右（与卡牌）通兑

---

## 一、货币体系（三层）

### 层 1：现实货币

| 项目 | 说明 |
|------|------|
| 币种 | USDC |
| 用途 | 后台结算用 |
| 说明 | 玩家充值、提现均以 USDC 结算 |

### 层 2：通兑货币（金币）

| 项目 | 说明 |
|------|------|
| 名称 | **金币** |
| 定位 | 中间层，可上下、左右通兑 |
| 兑换规则 | 1 USDC = 10 金币 |
| 充值赠送 | 充 10 USDC 送 10 金币（等于额外送 1 USDC 价值） |

**充值 UI 文案示例：**
> 充 10U 得 100 金币 + 送 10 金币

### 层 3：游戏积分

| 项目 | 说明 |
|------|------|
| 名称 | **积分** |
| 定位 | 游戏内产出，偏挖矿玩乐 |
| 兑换规则 | 10000 积分 = 1 金币 |
| 获取方式 | 通关、对局表现等 |

---

## 二、金币用途

### 1. 复活

| 项目 | 说明 |
|------|------|
| 场景 | 通关失败时 |
| 消耗 | 5 金币 |
| UI 文案 | 「花 5 金币，立即复活再来一局」 |
| 说明 | 小白一看就懂 |

### 2. 买额外卡槽

| 项目 | 说明 |
|------|------|
| 初始 | 7 个槽位 |
| 消耗 | 20 金币 = 永久增加 1 个槽位 |
| UI 文案 | 「多一个槽，通关率+30%（示意文案）」 |
| 说明 | 长线付费点，很直观 |

### 3. 抽奖

| 项目 | 说明 |
|------|------|
| 单抽 | 10 金币 抽 1 次 |
| 十连 | 90 金币 抽 10 次（送 1 次，等于 11 发） |

**抽奖交互：** 转盘固定，中心按钮 + 按钮外圈指针。用户按住按钮，松手后服务端返回结果角度，指针按按下时长旋转到该角度（按得越久转得越久，仅影响动画，不参与概率）。

**抽奖概率：** 10 档位，麒麟 1%、财神 0.01%、普通 8 档均分 98.99%。详见 [LOTTERY_DESIGN.md](./LOTTERY_DESIGN.md)。

---

## 三、卡牌系统（三层品质）

### 品质与爆率

| 品质 | 爆率 | 说明 |
|------|------|------|
| 普通卡 | 98% | 帮过关，不需要玩家理解更多 |
| 麒麟卡 | 1% | 与金币 1:1 通兑，幸运消费 |
| 财神卡 | 0.01%（抽奖） | 与 USDC 1:1 通兑，直接跟钱挂钩 |

> 抽奖页概率：麒麟 1%、财神 0.01%、普通 8 档均分 98.99%。见 [LOTTERY_DESIGN.md](./LOTTERY_DESIGN.md)。

### 普通卡

- **用途**：10 张普通卡 可兑换 1 张麒麟卡
- **定位**：只负责「帮过关」

### 麒麟卡

- **通兑**：与金币 1:1
- **定位**：幸运消费
- **效果（示例文案）：**
  - 「开局随机消一组」
  - 「多一个槽位（本局）」

### 财神卡

- **通兑**：每张财神卡 = 可兑换 1 USDC
- **定位**：直接跟钱挂钩
- **局内效果**：本局自动帮你摆放 3 张卡
- **玩家心智**：抽到财神卡 = 离提现更近了

## 四、通兑关系图

```
        USDC (层1)
           ↑↓ 1:10
        金币 (层2) ←→ 麒麟卡 (1:1)
           ↑↓ 10000:1

        财神卡 1:1 → USDC
```
